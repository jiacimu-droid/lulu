# 上下文与 token 调试说明

> 本页描述 `master` 当前真实实现。它用于定位上下文膨胀与调用场景问题，不保存用户的聊天正文或完整提示词。

## 入口

设置页的“API 控制台”展示最近的模型调用。所有经过 `GenerationHandler` 的角色生成都会记录：

- 调用场景与标题；
- 模型、提供者、流式状态和耗时；
- 模型返回的真实输入、输出、缓存与总 token；
- 本地估算的分层输入占比；
- 工具数量、工具名和主要 token 占用项；
- 完成、进行中或脱敏失败状态。

## 当前分层

- 人设/基础系统提示词；
- 工具定义与 schema；
- 工具额外提示词；
- 插件提示词注入；
- 跳过回复规则；
- 记忆、主动提醒等额外系统上下文；
- 历史对话上下文；
- 状态栏、学习状态和模板等变换器增量。

P2 的统一 `CompanionContextEnvelope` 完成后，额外系统上下文会继续细分为全局世界书、角色世界书、滚动摘要、记忆、关系/状态、承诺/关注等产品层；P0 已提供统一统计入口与场景标签，不需要再建立第二套日志系统。

## 隐私与保留

- 仅在 App 进程内保存最近 32 次调用；
- App 重启后自动清空；
- 日志对象只保留消息数量，不保留聊天正文、完整系统提示词或变换后的发送内容；
- 原始异常文本不会进入调试页，只显示脱敏失败状态；
- 清空按钮可随时立即移除当前进程中的全部记录；
- token 分层只保存标签、字符数与估算/真实 token 数量。

## 验收

1. 普通聊天发出一次请求后，控制台能看到“聊天”场景与分层 token。
2. 主动、电话等调用方传入各自 `ApiUsageSource` 后，控制台按场景区分。
3. 控制台中无法读取用户聊天正文、完整提示词或服务端回显的原始错误。
4. 连续超过 32 次调用后只保留最新 32 次；重启 App 后为空。
